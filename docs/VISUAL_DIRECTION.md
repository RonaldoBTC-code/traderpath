# Dirección visual: aventura educativa explorable

## Referencia

La referencia principal es *Alba: A Wildlife Adventure*: un mundo compacto, cálido y amable en el que el jugador aprende observando, recorriendo lugares e interactuando con personajes y objetos. TraderPath adopta esos principios de experiencia, no sus personajes, escenarios ni identidad visual.

## Principios

1. **El conocimiento vive en lugares.** Cada concepto se representa como un edificio, plaza, taller, puente o punto de observación.
2. **Primero explorar, después explicar.** La interfaz presenta una curiosidad visual antes de desplegar teoría.
3. **Una acción por encuentro.** Hablar, observar, ordenar, clasificar o decidir; nunca una pantalla cargada de controles.
4. **Ritmo tranquilo.** No se castiga al usuario por leer despacio. Los límites de tiempo solo aparecen cuando forman parte del aprendizaje.
5. **Mundo luminoso, datos precisos.** Las ciudades usan color y profundidad; los gráficos y cantidades conservan contraste y tipografía de datos.
6. **HUD mínimo.** Objetivo actual, progreso y acción disponible. El resto aparece bajo demanda.
7. **NPC como memoria.** Cada personaje representa una forma de pensar: método, contexto, FOMO, pánico o improvisación.
8. **Feedback visible en el mundo.** Completar una lección ilumina una ruta, restaura un edificio o abre un puente.

## Lenguaje visual de Ciudad Origen

- Cielo crema y naranja suave, vegetación verde y agua azul.
- Volúmenes redondeados y sombras amplias para sugerir un escenario low-poly.
- Caminos curvos que conectan hitos educativos.
- Bitcoin naranja como señalización, no como fondo dominante.
- Paneles oscuros únicamente para gráficas, datos y evaluación.
- Iconos grandes, textos cortos y objetivos visibles.

## Regla de implementación

Mientras TraderPath continúe como aplicación web 2D, cada ciudad se construirá como una vertical jugable con mapa ilustrado mediante HTML/CSS, puntos interactivos y transiciones. Una migración posterior a 3D deberá preservar contenido, progreso y lógica de evaluación como capas independientes del motor visual.
